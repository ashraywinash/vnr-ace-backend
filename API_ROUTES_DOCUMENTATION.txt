     "token_type": "bearer",
     "user": {
       "id": 1,
       "email": "admin@vnr.com",
       "role": "admin"
     }
   }

                            CLASSWORK MODULE
================================================================================

8. POST /classwork/chat
   Description: Natural language queries about student data (attendance, CGPA, backlogs, etc.)
   Content-Type: application/json
   Authentication: Uses mock user (no Authorization header required in current implementation)
   
   Required Inputs:
   {
     "message": "string (natural language query about student data)"
   }
   
   Supported Query Types:
   - Attendance queries: "Show me students with attendance below 75%"
   - CGPA queries: "What is the average CGPA of CSE students?"
   - Backlog queries: "Students with backlogs in year 2"
   - Branch/Section filters: "CSE students in section A"
   - Year filters: "All 3rd year students"
   - Combined queries: "ECE students with CGPA above 8.5 and attendance below 80%"
   
   Example Request Body:
   {
     "message": "Show me students with attendance below 75%"
   }
   
   OR
   
   {
     "message": "What is the average CGPA of CSE students?"
   }
   
   Response:
   {
     "reply": "AI-generated response with student data analysis (includes markdown tables and insights)",
     "metadata": {
       "record_count": 25,
       "query": "Show me students with attendance below 75%"
     }
   }
   
   Implementation Notes:
   - Data source: Excel file (data/student_data.xlsx)
   - Processing: LangGraph-based workflow with intent identification, data loading, and LLM processing
   - Graph state includes: user_query, role (mock: 4), context (user_id: 1)
   - Returns formatted markdown responses with tables and statistical analysis

17. POST /placements/shortlist
    Description: Shortlist students based on job description and criteria
    Content-Type: application/x-www-form-urlencoded OR multipart/form-data
    
    Required Inputs:
    - jd: string (job description)
    
    Optional Inputs:
    - min_gpa: float (minimum GPA requirement)
    - branch: string (branch filter: CSE, IT, ECE, etc.)
    
    Example Request Body:
    jd=Looking for software engineer with Python skills&min_gpa=8.0&branch=CSE
    
    Response:
    {
      "matches": [
        {"id": 1, "name": "Student A", "gpa": 9.0, "branch": "CSE", "match": "95%"},
        {"id": 4, "name": "Student D", "gpa": 8.8, "branch": "CSE", "match": "90%"}
      ]
    }

================================================================================
                          AUTHENTICATION NOTES
================================================================================

For protected endpoints (those requiring authentication):
1. First call POST /auth/login to get an access token
2. Include the token in subsequent requests:
   Header: Authorization: Bearer <your_access_token>

Role Hierarchy:
- admin (role_id: 1) - Full access
- faculty (role_id: 2 or 3) - Faculty-level access
- student (role_id: 4) - Student-level access
